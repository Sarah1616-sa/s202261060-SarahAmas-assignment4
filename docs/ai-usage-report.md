# AI Usage Report

## Tools Used

- OpenAI Codex for code refactoring, organization, UI polish suggestions, and documentation drafting
- AI-assisted text generation for improving project documentation structure and wording

## Use Cases

- Reorganizing the project into the required Assignment 4 folder structure
- Cleaning HTML, CSS, and JavaScript for readability and consistency
- Improving responsiveness and interface spacing without changing the core design
- Adding clear error handling for form validation and GitHub API states
- Drafting technical and project documentation in a more professional format

## Benefits

- Reduced time spent on repetitive cleanup tasks
- Helped identify structural issues such as missing folders and inconsistent documentation
- Provided fast iteration on user interface improvements
- Supported clearer technical writing for the submission documents

## Challenges

- AI suggestions still required manual review to ensure they matched assignment requirements
- Some recommendations needed simplification to avoid overcomplicating the portfolio
- Functional changes had to be checked carefully so the original behavior was not broken

## Learning Outcomes

- Improved understanding of how to structure a small front-end project professionally
- Better awareness of how to write maintainable JavaScript with clearer function responsibilities
- Stronger appreciation for validating AI output instead of accepting it directly
- More confidence in documenting technical decisions and project architecture

## Responsible Use

- AI was used as a support tool, not as a replacement for understanding the project
- All generated code and writing were reviewed, edited, and aligned with assignment goals
- The final portfolio content reflects my own background, projects, and academic work
- I can explain the implemented features, file structure, and project logic
